# 用户指南

BUPT GPA Calculator 是一个本地成绩档案工具。它不会登录教务系统，也不会把成绩写到用户目录或其他隐藏位置；所有成绩数据都保存在你选择的 `.db` 档案文件中。

## 30 秒快速开始

1. 启动程序，选择“新建成绩档案”，把 `.db` 文件保存到你希望长期保管的位置；
2. 点击“新增学生”，填写学号和姓名；
3. 进入“课程记录”页，点击“导入教务成绩”；
4. 打开北邮教务系统成绩页，在成绩表格内部点击一次，按 `Ctrl+A` 全选表格，再按 `Ctrl+C` 复制；
5. 回到导入窗口，点击“识别成绩表”，核对预览后确认导入；
6. 回到“成绩概览”页查看 GA、GPA、已修总学分和最新学期 GA。

## 成绩档案

成绩档案是一个独立的 `.db` 文件。一个档案可以保存多个学生，程序以学号作为唯一标识。你可以随时打开已有档案、另存当前档案，或把 `.db` 文件复制到其他位置进行备份。

程序不会自动创建云同步、用户目录配置或隐藏数据目录。请自行保管 `.db` 文件；如果删除它，对应的成绩档案也会被删除。

## 学生管理

点击顶部的“新增学生”可以在当前档案中添加学生。点击“管理”可以查看和修改学生信息。学号作为唯一标识不可修改；如果学号填错，建议删除后重新添加学生。

顶部学生下拉框用于在同一档案内切换不同学生。切换学生前，建议先保存当前档案。

## 录入课程

课程记录包含：

- 开课学期，例如 `2025-2026-2`；
- 课程编号，可为空；
- 课程名称；
- 整数成绩；
- 学分，可为 `0`；
- 是否计入 GA / GPA。

来源列用于标记课程来源：

- “导入”：来自教务系统表格；
- “修改”：原本来自教务导入，但之后被手动编辑过；
- “手动”：由用户手动新增。

计入列用绿色圆点表示参与计算，灰色圆点表示不参与计算。少数有成绩和学分、但不计入最终总成绩的课程，可以在新增或编辑课程时取消“计入 GPA 与 GA”。

## 导入北邮教务成绩

1. 打开北邮教务系统的“课程成绩”页面并查询成绩；
2. 在成绩表格内部点击一次；
3. 按 `Ctrl+A` 全选成绩表格，再按 `Ctrl+C` 复制完整表格，包含表头；
4. 在程序的“课程记录”页选择“导入教务成绩”；
5. 点击“识别成绩表”；如果文本框为空，程序会读取剪贴板并解析；也可以先手动粘贴再识别；
6. 核对预览结果后确认导入。

程序按表头识别开课学期、课程编号、课程名称、成绩和学分，不依赖固定列顺序。重复导入同一学期成绩时，会按“学期 + 课程编号”更新已有课程，并重建为本次教务表中的课程顺序；无编号课程仅在同学期、同名且唯一时合并。

这个设计适合“成绩逐步公布”的情况：如果昨天导入了 3 门课，今天教务系统显示 5 门课，再复制完整表格导入一次即可。程序会更新重复课程，并按教务系统当前表格顺序保存课程。

## 成绩概览

“成绩概览”页显示四个核心结果：

- GPA：按计入课程的学分加权绩点；
- GA：按计入课程的学分加权成绩；
- 已修总学分：当前范围内所有已录入课程的学分总和；
- 最新学期 GA：固定显示全部课程中最新学期的 GA。

右上角可以选择全部学期、某一学年或单个学期。选择学年或单学期后，前三张卡片会随范围变化；第四张“最新学期 GA”会变灰，避免和当前筛选范围混淆。

左侧趋势图默认显示 GA，点击图表右上角的切换按钮可以在 GA 和 GPA 趋势之间切换。

## 排序与筛选

“课程记录”页支持按课程名称、编号或学期搜索，也可以按学期和计入状态筛选。

点击课程表任一列标题，会依次切换升序、降序和默认顺序。默认顺序按学期排列；同一学期内先显示教务导入课程，并保留教务表顺序，再显示手动课程。

## 计算规则

只有标记为“计入”的课程参与 GA 和 GPA 计算。0 学分课程会保留在课程列表中，但不会改变加权结果。GA 与 GPA 显示四位小数，学分显示一位小数。

“计算规则”页可以查看百分制成绩与绩点的对照关系，也可以查看当前课程逐项参与计算的明细。

## 下载与运行

Release 页面提供 Windows x64 压缩包：

```text
BUPT-GPA-vX.Y.Z-win-x64.zip
```

下载后解压，运行其中的 `BUPT-GPA.exe`。这个版本是 self-contained portable 程序，已经内置运行所需的 .NET 组件，不需要额外安装运行环境。

## 常见问题

### 程序会自动访问教务系统吗？

不会。程序只识别你复制到剪贴板或粘贴到导入窗口里的成绩表文本，不会保存账号密码，也不会向教务系统发起请求。

### 为什么某门课没有影响 GA / GPA？

请检查课程是否被标记为“计入”。如果课程学分为 `0`，它会保留在列表中，但不会改变加权结果。

### 为什么导入后课程顺序变化了？

教务导入课程会按教务系统当前表格顺序保存。成绩逐步公布时，教务系统表格顺序可能会随着新成绩出现而补齐；重复导入完整表格即可同步到最新顺序。

### 数据保存在哪里？

数据保存在你选择的 `.db` 档案文件中。程序不会把成绩数据写到其他隐藏目录。
